# Template Authoring

Templates are React components that render from `ProfileDocument`. They should not hard-code sample content.

## Resume Templates

Add metadata to `resumeTemplateMetas` and a component to `resumeTemplateComponents` in `src/templates/resumeTemplates.tsx`.

Rules:

- Respect `settings.sectionLabels`.
- Respect `settings.hiddenSections`.
- Respect `settings.sectionOrder` when the layout is linear.
- Use `settings.themeColor` for accent color.
- Keep A4 sizing through the `resume-page` class.
- Avoid adding template ids that already exist.

## Portfolio Templates

Add metadata and a component in `src/templates/portfolioTemplates.tsx`.

Portfolio templates should highlight:

- selected projects
- tech stack
- links to repo/demo/video
- contact links
- concise summary

Static export is generated by `src/lib/portfolioExport.ts`. If a new template needs special static HTML, add a branch there using the same `templateId`.

## Printable Portfolio Deck Templates

Add printable Deck metadata and components in `src/templates/portfolioDeckTemplates.tsx`. Deck components should render through the shared `PortfolioDeckView` sequencing and content primitives unless a new page type is required.

Rules:

- Register the template id in `portfolioDeckTemplateIds` and `PortfolioDeckTemplateId`.
- Keep Deck colors independent from CV `settings.themeColor`.
- Provide an accessible recommended palette in Deck template metadata; selecting a template applies that palette.
- Scope template CSS under `deck-template-<id>`.
- Preserve A4 dimensions and prevent page overflow.
- Preserve the shared semantic DOM reading order even when visual layouts differ.
- Resolve images through the supplied `assets` map; do not load remote URLs directly from Playwright.
- Add registry, contrast, HTML rendering, accessibility, and Playwright workflow coverage.
